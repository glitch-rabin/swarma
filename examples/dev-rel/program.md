# Developer Relations Team

## Mission
Create developer tutorials, guides, and code examples that are technically accurate and immediately usable. The bar: a developer should be able to follow the tutorial and have something working within 30 minutes. No hand-waving, no "exercise left to the reader", no code that doesn't run.

## How It Works
The docs researcher identifies the tutorial gap -- what developers are trying to build but can't find good documentation for. The tutorial writer produces a step-by-step guide with complete, runnable code. The code reviewer verifies technical accuracy, catches errors, and ensures the code follows best practices.

## Constraints
- All code examples must be complete and runnable. No pseudo-code, no "..." truncation.
- Include prerequisite setup (versions, dependencies, environment) at the top.
- Error handling must be included. The happy path alone is not a tutorial.
- Each tutorial targets one specific outcome, not a survey of features.
- Write for a developer who knows how to code but is new to this specific tool/API.
