# Hook Lab

## Mission

Test messaging angles systematically. Find which hooks, frames, and structures make people stop, read, and save. Every run is an experiment -- not content production.

## How Growth Teams Actually Do This

A hook lab is the messaging R&D arm of a growth team. Instead of guessing what resonates, you run controlled variations:

- **Angle testing**: same topic, different entry point (pain vs aspiration, data vs story, contrarian vs consensus)
- **Structure testing**: same angle, different format (question vs statement, short vs long, numbered vs narrative)
- **Proof testing**: same claim, different evidence (metric vs anecdote, before/after vs testimonial)

The experiment loop makes this automatic. Each run produces a variation, scores it, and the strategy file evolves with what actually works.

## Experiment Patterns

Real hook experiments this squad should run:

1. **Loss framing vs gain framing** -- "you're losing $X" vs "you could save $X"
2. **Specific numbers vs vague claims** -- "47% of teams" vs "most teams"
3. **Story lead vs data lead** -- "last tuesday our deploy broke" vs "deploy failures cost $4.2M/yr"
4. **Contrarian vs consensus** -- "everyone's wrong about X" vs "the best teams do X"
5. **Question hook vs statement hook** -- "why do 80% of launches fail?" vs "80% of launches fail."
6. **Personal vs universal** -- "I lost $50K" vs "companies lose $50K"

## Metrics

- **Primary**: hook_score (self-eval on pattern interrupt + specificity + emotional pull)
- **Production**: save_rate, dwell_time, click-through rate via webhook
- **Signal**: a high hook score with low save rate means the hook is clickbait, not substance. optimize for saves.

## Constraints

- Every hook must be testable against a clear baseline
- No "thought leadership" energy. practitioner voice only.
- One variable per experiment. If you change the angle AND the structure, you learn nothing.
