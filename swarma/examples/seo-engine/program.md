# SEO Engine

## Mission

Produce search-optimized content through systematic keyword targeting and quality experimentation. Every piece targets a specific query with measurable ranking potential.

## How Growth Teams Actually Do This

SEO teams that win don't publish volume -- they run experiments:

- **Keyword clustering**: group related queries into content hubs instead of one-off pages
- **Content format testing**: listicle vs guide vs comparison vs tool page for the same keyword
- **SERP-aware writing**: study what's ranking, find the gap, fill it better
- **Programmatic SEO**: templatized content at scale for long-tail patterns (e.g., "X vs Y" comparisons)

The experiment loop tests which formats, structures, and angles produce content that self-evaluates highest on ranking potential before you even publish.

## Experiment Patterns

1. **Format wars** -- "10 best X" listicle vs "complete guide to X" vs "X vs Y comparison"
2. **Depth vs breadth** -- 3000-word deep dive vs 800-word focused answer
3. **Structure testing** -- TL;DR up top + detail below vs progressive disclosure
4. **LLMO optimization** -- tables for data, FAQ sections, conversational tone (LLM-optimized content)
5. **Freshness signals** -- dated content vs evergreen vs "updated March 2026"
6. **Internal linking density** -- 3 links vs 8 links vs topic cluster approach

## Metrics

- **Primary**: seo_quality (self-eval on keyword targeting + depth + structure + LLMO readiness)
- **Production**: organic_impressions, click_through_rate, average_position via webhook
- **Signal**: high quality score + low CTR = title/meta description problem, not content problem.

## Constraints

- Every piece must target a specific keyword cluster, not a vague topic.
- Include structured data opportunities (FAQ, how-to, comparison tables).
- LLMO-ready: TL;DR early, tables for data, conversational, FAQ section.
- No thin content. If you can't add value beyond what's already ranking, skip the keyword.
