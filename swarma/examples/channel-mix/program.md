# Channel Mix

## Mission

Test how the same insight performs across different distribution channels. Cross-posting is the baseline. Platform-native adaptation is the experiment.

## How Growth Teams Actually Do This

Most teams either: (a) post the same thing everywhere, or (b) only use one channel. Both leave distribution on the table.

A channel-mix squad takes one insight and runs it through platform-specific filters:

- **LinkedIn**: professional frame, save-worthy, longer-form, no links in post body
- **Twitter/X**: raw, opinionated, quote-tweet-able, reply-bait
- **Email**: personal, direct, "I'm telling you this" energy, clear CTA

The experiment loop compares which platform-content combinations drive the best engagement-to-effort ratio.

## Experiment Patterns

1. **Native vs cross-posted** -- same exact text vs platform-adapted. measure engagement delta.
2. **Angle rotation** -- data angle on LinkedIn, hot take on Twitter, story in email. vs opposite.
3. **Timing experiments** -- same content, different posting windows per platform.
4. **Format testing** -- text-only vs carousel (LinkedIn), thread vs single tweet (Twitter), plain vs designed (email).
5. **Depth gradient** -- teaser on Twitter, full take on LinkedIn, deep dive in email.

## Metrics

- **Primary**: platform_engagement (self-eval on native feel + value density)
- **Production**: per-platform saves, clicks, replies via webhook
- **Signal**: high engagement on one platform + low on another = the angle works, the format doesn't. Isolate format from message.

## Constraints

- Never cross-post verbatim. That's the control group, not the strategy.
- Each platform output must feel native -- someone shouldn't be able to tell it was adapted.
- Track channel-specific metrics separately. Aggregate numbers hide what's working.
